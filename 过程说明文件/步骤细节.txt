1.第四模块包含的页面和能力:
需要有两个小页面：第一个页面用于摆放预测功能入口，从已经添加进入数据库的motor_insurance中自由选择1到10条保单信息进行预测，推理结束后即刻展示理赔概率、风险等级、预计理赔金额。同时该页面需要存有历史预测信息，包括ID，理赔概率、风险等级、预计理赔金额这四项信息，这个信息应支持人工删除，但不能修改或者人工添加。第二个页面用于统计历史预测信息，展示已有历史预测信息中的风险等级分布、预计理赔金额分布（按照500元为步长的频率分布直方图）。

2.模型在线推理需要的输入字段：
在线推理模型的输入将从数据库中的motor_insurance获取，获取后再根据训练时的模型输入来截取需要的字段来在线推理。

3.预测历史是否要由 Spring Boot 落库:需要落库，表的名字为insur_pred，沿用ID作为业务主管联键，需要保存prediction_time、 model_version、 threshold_used

4.我希望 Spring Boot 调 FastAPI 后，返回给前端的是：先落库，再返回数据库中预测记录

5.FastAPI 服务端口：端口号没有特殊要求，用你的默认即可

6.第一个预测页面里，Spring Boot 侧我希望怎么发起预测：前端传 1~10 个保单 ID 给 Spring Boot，由 Spring Boot 自己查 motor_insurance 再调 FastAPI

7.历史预测记录删除时，我希望是：按预测记录主键删除

8.统计页的数据范围，我希望是：全部历史预测记录，记录来自于insur_pred表

9.风险等级是否保持我现在写死的规则不变：保持不变即可

10.第一个页面的交互形式:从 motor_insurance 列表里勾选 1~10 条

11.预测结果展示位置:当前页结果表格(弹窗展示) + 历史记录表格

12.历史预测信息的分页规则：每页显示10条

13.预测页是否需要筛选条件：我不需要筛选可选保单

14.预测结果页显示哪些列：可以顺手显示预测时间和模型版本

15.风险等级中文显示：前端显示中文

16.第二个统计页的图表样式：风险等级分布用饼图，金额分布用柱状图

17.菜单和路由名字：按照你的建议来就行

18.预测页是否允许重复预测同一个保单：允许重复预测，但是如果已预测过就提示，确认真的要重复预测后再预测
