2）主干网络的结构设计
主干网络Backbone是当前系统分类模型的核心部分，其作用是将结构化输入特征映射为统一的高层表示向量。与普通MLP直接逐层堆叠不同，当前系统采用“输入投影层 + 四个残差块”的共享主干结构，并仅服务于单任务二分类，即预测保单在当前年度是否发生理赔。结合系统当前实际加载的模型权重，主干网络的隐藏层宽度依次为128、256、256、128和128，最终输出维度固定为128，该向量作为单一分类头的输入。
（1）输入投影层设计：输入投影层采用“Linear(input_dim, 128) + LayerNorm(128) + GELU + Dropout”的组合形式，将原始特征首先映射到统一的128维表示空间，为后续残差块提供稳定的特征基础。在当前系统配置中，主干部分的dropout取值为0.20，而模型输入端额外的input_dropout设为0.0，因此实际生效的随机失活主要发生在主干内部和分类头内部。线性映射后的LayerNorm有助于缓解不同特征维度数值尺度差异造成的训练波动，GELU则进一步提升了模型对非线性关系的表达能力。
（2）残差块设计：ResidualBlock是当前模型最重要的结构单元。单个残差块内部的计算流程为：输入向量首先经过第一层全连接映射，再依次通过LayerNorm与GELU完成归一化和非线性激活，随后施加Dropout；之后再经过第二层全连接和第二个LayerNorm，最后与捷径分支相加，并再次通过GELU输出。该结构既能够提升深层网络的表达能力，又能够通过残差连接缓解梯度传播困难和宽度切换带来的优化不稳定问题。
\[ h = \mathrm{GELU}\!\left(\mathrm{LN}_2\!\left(W_2\!\left(\mathrm{Dropout}\!\left(\mathrm{GELU}\!\left(\mathrm{LN}_1(W_1x)\right)\right)\right)\right) + S(x)\right) \]
其中，S(x)表示捷径分支。当输入维度与当前残差块输出维度一致时，S(x)取恒等映射；当两者不一致时，代码中通过无偏置线性投影层nn.Linear(in_dim, hidden_dim, bias=False)完成维度对齐。这样设计的好处在于，即使网络宽度发生变化，也能够保证主分支与捷径分支在相加时具有一致的张量形状。当前系统实际代码如下：
class ResidualBlock(nn.Module):
    def __init__(self, in_dim: int, hidden_dim: int, dropout: float = 0.20):
        super().__init__()
        self.fc1 = nn.Linear(in_dim, hidden_dim)
        self.norm1 = nn.LayerNorm(hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, hidden_dim)
        self.norm2 = nn.LayerNorm(hidden_dim)
        self.drop = nn.Dropout(dropout)
        self.act = nn.GELU()
        self.shortcut = (
            nn.Linear(in_dim, hidden_dim, bias=False)
            if in_dim != hidden_dim
            else nn.Identity()
        )
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        residual = self.shortcut(x)
        out = self.act(self.norm1(self.fc1(x)))
        out = self.drop(out)
        out = self.norm2(self.fc2(out))
        return self.act(out + residual)
（3）主干宽度设计：当前系统实际部署模型的共享主干宽度变化依次为128→256、256→256、256→128和128→128，结合输入投影层后可视为一条128→256→256→128→128的分层表示路径。该结构仍然保留了“先升维、再提炼、后压缩、再精炼”的设计思想：模型首先从128维升到256维，以扩大特征交互空间；随后在256维空间中继续深化特征提取；再将表示压缩回128维，以过滤冗余信息；最后在128维空间中完成紧凑表征的进一步精炼。相比旧版的256→512→512→256→256主干，当前结构整体更轻量，更适合现阶段系统中的单任务分类部署场景。
3）分类输出头设计
分类头ClassificationHead的作用是预测保单在当前年度是否发生理赔。当前系统中，该分支首先将共享表示从128维压缩到32维，以提取更适用于分类决策的紧凑特征；随后经过LayerNorm和GELU提升特征的稳定性与非线性区分能力；再通过Dropout抑制过拟合；最后由线性层输出一个标量logit。代码中采用squeeze(-1)将输出张量形状压缩为(B,)，即对一个批次中的每个样本输出一个标量。
需要强调的是，当前模型不仅保留了单一分类头，还在forward_logits中设置了head_samples=10。也就是说，当head_samples大于1时，分类头会对同一共享表示执行多次前向计算，并对logit结果取平均，从而增强分类输出的平滑性与稳定性。在训练阶段，这一设计可以结合分类头中的Dropout形成近似的头部采样集成；在推理阶段，模型仍然输出单一logit，并继续使用BCEWithLogitsLoss对应的数值稳定形式，而不在网络内部显式调用Sigmoid。
4）损失函数设计
当前系统采用的损失函数是带正类权重的二元交叉熵损失，即BCEWithLogitsLoss，并且当前版本仅保留单任务分类目标，不再包含旧版文档中涉及的回归分支、双任务头或动态任务权重。设正类权重参数为pos_weight，其作用是当样本为正类时，对应损失项被放大pos_weight倍，从而提升少数类样本在梯度更新中的贡献。在当前系统配置中，pos_weight设置为2.50，用于抑制模型在类别不均衡条件下过度偏向“无理赔”类别。
与旧版设计不同，当前系统虽然在损失配置中保留了label_smoothing参数，但其实际取值为0.0，因此标签平滑并未在当前部署模型中生效。换言之，当前真正参与训练优化的是“单任务分类损失 + 正类权重补偿”这一组合，而不是“正类加权与标签平滑同时启用”的方案。
\[ \mathcal{L}_{\mathrm{BCE}} = - \left[ w_{+}\, y \log\!\left(\sigma(z)\right) + (1-y)\log\!\left(1-\sigma(z)\right) \right] \]
其中，z表示模型输出的原始logit，y表示真实类别标签，σ(·)表示Sigmoid函数，w_{+}表示正类损失权重系数。由于代码直接使用BCEWithLogitsLoss实现，Sigmoid运算和交叉熵计算被合并在一个数值稳定的操作过程中，从而能够避免概率接近0或1时出现数值下溢或梯度不稳定问题。
4.3训练策略设计
针对车险理赔预测任务中常见的类别分布不均衡、训练初期优化不稳定以及深度模型易出现过拟合等问题，当前系统在训练阶段实际生效的策略主要包括：基于验证集的自动阈值搜索与召回率约束策略、基于AdamW和Cosine Warmup的学习率调度策略，以及基于AUC监控指标的Early Stopping策略。此外，训练流程还启用了AMP混合精度和梯度裁剪，以进一步提升训练效率与数值稳定性。需要特别说明的是，虽然数据加载代码中保留了不平衡采样的实现接口，但当前系统配置balanced_sampling=False，且现部署模型并未启用WeightedRandomSampler，因此不平衡采样不属于当前真正生效的训练策略。
1）自动阈值搜索与召回率约束策略
当前系统在每个epoch完成验证集评估后，不直接固定使用0.5作为分类阈值，而是在0.05到0.95之间均匀生成200个候选阈值，并逐一计算各阈值下的Precision、Recall以及目标评分。在当前配置下，threshold_metric设置为f1，threshold_beta设置为1.3，因此阈值搜索实际优化的是更偏重召回率的F_β指标，而不是普通的标准F1。与此同时，系统还设置了threshold_min_recall=0.830，要求所选阈值尽量满足召回率下限约束。
该策略的意义在于，它将“排序能力”和“业务判定规则”分离开来：模型本身负责输出理赔概率，而最终的正负类划分则通过验证集阈值搜索确定。若存在满足Recall下限的可行阈值，则系统在可行集合内优先选择F_β得分最高的阈值；若不存在满足条件的候选阈值，则退化为在全体候选中优先选择Recall更高的方案。这样设计能够使模型输出更贴合车险风控场景下“宁可适度增加误报，也要尽量避免漏判高风险保单”的业务目标。当前系统实际加载的best_threshold.pt中保存的分类阈值约为0.5656。
2）Cosine Warmup学习率调度与优化策略
为了提高训练初期的稳定性，并在中后期实现更加平滑和有效的参数优化，当前系统采用AdamW作为优化器，初始学习率设置为1e-4，同时配合Cosine Warmup学习率调度策略。在实现上，该策略由两部分组成：前期使用LinearLR完成线性预热，预热轮数为5；随后切换到CosineAnnealingLR进行余弦退火，并将最小学习率控制在1e-6。二者通过SequentialLR按照epoch级别顺序衔接。
从训练策略整体设计来看，Cosine Warmup的价值在于它将训练过程划分为“稳定启动”和“平滑收敛”两个阶段：在训练初期避免学习率过大导致参数震荡，在训练后期逐步降低学习率以细化参数搜索。同时，当前系统还启用了AMP混合精度训练，并设置grad_clip=1.0进行梯度裁剪，从而在保证训练速度的同时进一步降低梯度爆炸和数值不稳定风险。这种组合策略尤其适合特征维度较多、类别分布不均衡的保险表格分类任务。
3）Early Stopping及监控指标设计
为避免模型在训练后期对训练集持续拟合而导致泛化性能下降，当前系统采用Early Stopping策略作为训练终止机制。在具体实现上，系统设置patience=8、min_delta=5e-5，并在验证集性能在若干轮内未出现足够幅度改进时提前终止训练。该机制能够在模型已经学习到主要判别模式后及时停止训练，从而减少继续迭代对噪声和局部模式的过拟合。
在监控指标选择上，当前系统将early_stop_metric设置为auc。选择AUC作为早停监控指标具有明确的方法论依据。首先，AUC刻画的是模型对正负样本的整体排序能力，相较于Accuracy、Precision或固定阈值下的F1，它对类别不均衡问题更加稳健。其次，训练流程中已经启用了自动阈值搜索和召回率约束，不同epoch的最佳分类阈值可能发生变化；若直接以Precision、Recall或F1作为唯一早停依据，容易将“阈值变化带来的波动”与“模型本身排序能力变化”混杂在一起。因此，使用AUC负责监控模型排序质量，再由阈值搜索模块负责生成最终分类阈值，是当前系统中更合理、也更稳定的训练设计。
